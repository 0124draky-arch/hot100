## 1. 哈希表模式

### 1.1 两数之和模式（查找配对）
**适用场景：** 需要找两个元素满足某种关系（和、差、积等）

**核心思路：** 边遍历边查找，用哈希表存储"已经遍历过的元素"

```javascript
function twoSum(nums, target) {
    const map = new Map(); // 存储 值 -> 索引
    for (let i = 0; i < nums.length; i++) {
        const complement = target - nums[i]; // 需要找的另一个数
        if (map.has(complement)) {
            return [map.get(complement), i];
        }
        map.set(nums[i], i); // 把当前数存入
    }
}
```

**时间复杂度：** O(n)  
**空间复杂度：** O(n)

### 1.3 Set 起点判断模式（最长连续序列）
**适用场景：** 判断元素是否存在、去重、连续序列问题

**核心思路：** 用 Set 快速判断数字是否存在，只从“真正的起点”开始统计，避免重复计算。

```javascript
function longestConsecutive(nums) {
    const set = new Set(nums);
    let maxLen = 0;

    for (const num of set) {
        if (!set.has(num - 1)) {
            let current = num;
            let len = 0;

            while (set.has(current)) {
                current++;
                len++;
            }

            maxLen = Math.max(maxLen, len);
        }
    }

    return maxLen;
}
```

**关键点：**
- `Set` 只关心元素在不在，不需要存键值对
- `!set.has(num - 1)` 表示 `num` 是一段连续序列的起点
- 只从起点往后数，每段序列只统计一次

**时间复杂度：** O(n)  
**空间复杂度：** O(n)

### 1.4 前缀和 + 哈希表
**适用场景：** 子数组和问题、连续子数组问题

**核心思路：** 前缀和 + 哈希表存储"前缀和出现的次数"

```javascript
function subarraySum(nums, k) {
    const map = new Map();
    map.set(0, 1);

    let sum = 0;
    let count = 0;

    for (const num of nums) {
        sum += num;
        if (map.has(sum - k)) {
            count += map.get(sum - k);
        }
        map.set(sum, (map.get(sum) || 0) + 1);
    }

    return count;
}
```

**关键点：**
- `sum - k` 的含义：如果存在这个前缀和，那么中间部分的和就是 k
- 初始化 `map.set(0, 1)` 处理从头开始的子数组

## 2. 数组模式

### 2.1 前缀积 + 后缀滚动乘积
**适用场景：** 需要计算“除当前位置以外”的乘积，且不能使用除法

**核心思路：** `answer` 先存每个位置左边的乘积，再从右往左用变量 `right` 滚动维护右边乘积。

```javascript
function productExceptSelf(nums) {
    const n = nums.length;
    const answer = new Array(n);

    answer[0] = 1;
    for (let i = 1; i < n; i++) {
        answer[i] = answer[i - 1] * nums[i - 1];
    }

    let right = 1;
    for (let i = n - 1; i >= 0; i--) {
        answer[i] *= right;
        right *= nums[i];
    }

    return answer;
}
```

**关键点：**
- 第一遍结束后，`answer[i]` 表示 `nums[i]` 左边所有数的乘积
- 第二遍处理 `i` 前，`right` 表示 `nums[i]` 右边所有数的乘积
- 必须先 `answer[i] *= right`，再 `right *= nums[i]`，否则会把当前位置自己算进去

**时间复杂度：** O(n)  
**额外空间复杂度：** O(1)，输出数组 `answer` 不计入额外空间

### 2.2 数组中的快慢指针找环
**适用场景：** 数组长度为 `n + 1`，数字范围为 `1 ~ n`，要求不修改数组并用 O(1) 额外空间找重复数。

**核心思路：** 把数组看成“下标指向 `nums[i]` 的隐式链表”，重复数字就是环入口。

```javascript
function findDuplicate(nums) {
    let slow = 0;
    let fast = 0;

    do {
        slow = nums[slow];
        fast = nums[nums[fast]];
    } while (slow !== fast);

    let ptr1 = 0;
    let ptr2 = slow;

    while (ptr1 !== ptr2) {
        ptr1 = nums[ptr1];
        ptr2 = nums[ptr2];
    }

    return ptr1;
}
```

**关键点：**
- `nums[i]` 表示从下标 `i` 下一步跳到哪个下标
- 数字范围是 `1 ~ n`，所以不会跳回下标 `0`，`0` 只是进入链表的起点
- `fast = nums[nums[fast]]` 表示快指针连续跳两步
- 第一阶段相遇后，`slow` 和 `fast` 都可以作为相遇点
- 第二阶段一个指针从 `0` 出发，另一个从相遇点出发，同速走，再次相遇就是环入口

**时间复杂度：** O(n)  
**空间复杂度：** O(1)

### 287. 寻找重复数
**难度：** 中等 ⭐⭐⭐  
**完成日期：** 2026-06-15

**题目：** 给定一个包含 `n + 1` 个整数的数组 `nums`，其中每个整数都在 `1 ~ n` 之间，找出唯一重复的数字。要求不能修改数组，并且只能使用 O(1) 的额外空间。

**核心思路：**
- 把数组看成“下标指向 `nums[i]` 的隐式链表”
- 因为值域是 `1 ~ n`，所以从 `0` 出发一定会进入 `1 ~ n` 中的某个环
- 重复数字对应的就是环入口
- 第一阶段用快慢指针在环里相遇，第二阶段一个指针回到起点，另一个留在相遇点，同速前进，最终相遇点就是环入口

**代码：**
```javascript
var findDuplicate = function(nums) {
    let slow = 0;
    let fast = 0;

    do {
        slow = nums[slow];
        fast = nums[nums[fast]];
    } while (slow !== fast);

    let ptr1 = 0;
    let ptr2 = slow;

    while (ptr1 !== ptr2) {
        ptr1 = nums[ptr1];
        ptr2 = nums[ptr2];
    }

    return ptr1;
};
```

**时间复杂度：** O(n)  
**空间复杂度：** O(1)

**易错点：**
- `while (slow !== fast)` 不能直接用在第一阶段，因为初始时两者都等于 0，循环不会执行
- `fast = nums[nums[fast]]` 才表示快指针走两步
- 第二阶段里 `ptr1` 和 `ptr2` 每次都只能走一步，不能再让快指针走两步
- `ptr2` 可以写成 `slow` 或 `fast`，因为第一阶段结束时它们已经相遇

**学习心得：**
- 这个题最难的不是代码，而是把数组理解成“传送门”模型
- `nums[i]` 不是普通的数值，而是“下一步要跳到的下标”
- 重复数字之所以是环入口，是因为有两个不同下标指向同一个下标
- 用 `Set` 也能找重复数，但额外空间是 O(n)，不满足题目要求

## 3. 双指针模式

### 2.1 对撞指针（两端向中间）
**适用场景：** 有序数组、需要从两端查找、最大面积类问题

**核心思想：** 从两端出发，根据当前状态决定移动哪一侧，逐步逼近最优解

```javascript
// 通用模板
function twoPointers(arr) {
    let left = 0, right = arr.length - 1;
    while (left < right) {
        if (满足条件) {
            // 处理逻辑
            left++;
            right--;
        } else if (需要增大) {
            left++;
        } else {
            right--;
        }
    }
}
```

**贪心变体（如 LC11 盛最多水）：**  
> 面积由短板决定，每次移动较短的一边
```javascript
// LC11 对撞指针 + 贪心
function maxArea(height) {
    let left = 0, right = height.length - 1;
    let max = 0;
    while (left < right) {
        const area = (right - left) * Math.min(height[left], height[right]);
        max = Math.max(max, area);
        // 移动较短的那边：移动长板只会让宽度减小且短板不变，面积不可能变大
        if (height[left] >= height[right]) right--;
        else left++;
    }
    return max;
}
```

**时间复杂度：** O(n)  
**空间复杂度：** O(1)

**三数之和变体（排序 + 固定i + 去重）：**
> 固定第一个数，对撞指针找剩余两数，三处用 `while` 去重

```javascript
// LC15 三数之和模板
function threeSum(nums) {
    const result = [];
    nums.sort((a, b) => a - b); // 必须先排序

    for (let i = 0; i < nums.length - 2; i++) {
        if (i > 0 && nums[i] === nums[i - 1]) continue; // i 去重
        if (nums[i] > 0) break; // 最小三数 > 0，无解

        let left = i + 1, right = nums.length - 1;
        while (left < right) {
            const sum = nums[i] + nums[left] + nums[right];
            if (sum === 0) {
                result.push([nums[i], nums[left], nums[right]]);
                while (left < right && nums[left] === nums[left + 1]) left++;   // left 去重
                while (left < right && nums[right] === nums[right - 1]) right--; // right 去重
                left++; right--;
            } else if (sum < 0) {
                left++;
            } else {
                right--;
            }
        }
    }
    return result;
}
```

**去重关键：** 用 `while` 不用 `if`，跳过所有连续重复值；重复的 `i` 搜索空间是前一个的子集，跳过不漏答案。

**时间复杂度：** O(n²)



**接雨水变体（双指针，O(1) 空间）：**

> 哪边 max 小，那边水量已确定，安全结算后移动

```javascript
// LC42 接雨水双指针模板
function trap(height) {
    let left = 0, right = height.length - 1;
    let leftMax = height[0], rightMax = height[right];
    let water = 0;

    while (left < right) {
        if (leftMax >= rightMax) {
            water += rightMax - height[right];
            right--;
            rightMax = Math.max(rightMax, height[right]);
        } else {
            water += leftMax - height[left];
            left++;
            leftMax = Math.max(leftMax, height[left]);
        }
    }
    return water;
}
```

**关键：** `leftMax < rightMax` 时，左边的水量上限是 `leftMax`（右边够高，min 由左决定），可直接结算。

### 2.3 滑动窗口（重要！）
**适用场景：** 子串/子数组问题、连续区间问题

**核心思路：** 维护一个动态窗口，右指针扩展，左指针收缩

```javascript
function slidingWindow(s) {
    const map = new Map(); // 窗口内元素统计
    let left = 0, right = 0;
    let result = 初始值;
    
    while (right < s.length) {
        // 1. 扩大窗口：right指针移动，加入新元素
        const c = s[right];
        right++;
        map.set(c, (map.get(c) || 0) + 1);
        
        // 2. 判断是否需要收缩窗口
        while (窗口需要收缩的条件) {
            // 3. 缩小窗口：left指针移动，移除元素
            const d = s[left];
            left++;
            map.set(d, map.get(d) - 1);
        }
        
        // 4. 更新结果
        result = Math.max(result, right - left);
    }
    return result;
}
```

**滑动窗口四步法：**
1. 右指针扩展：加入新元素
2. 判断收缩：窗口不满足条件时收缩
3. 左指针收缩：移除左边元素
4. 更新答案：在合适的时机更新结果

**一句话模板：** 右指针负责探索新元素，左指针负责修复窗口；窗口不合法时移动左指针，合法时更新答案。

#### 无重复字符窗口
**适用场景：** 找最长不含重复字符的连续子串。

**Set 写法：**
```javascript
function lengthOfLongestSubstring(s) {
    const set = new Set();
    let left = 0;
    let maxLen = 0;

    for (let right = 0; right < s.length; right++) {
        while (set.has(s[right])) {
            set.delete(s[left]);
            left++;
        }

        set.add(s[right]);
        maxLen = Math.max(maxLen, right - left + 1);
    }

    return maxLen;
}
```

**Map 跳跃写法：**
```javascript
function lengthOfLongestSubstring(s) {
    const map = new Map();
    let left = 0;
    let maxLen = 0;

    for (let right = 0; right < s.length; right++) {
        const char = s[right];

        if (map.has(char) && map.get(char) >= left) {
            left = map.get(char) + 1;
        }

        map.set(char, right);
        maxLen = Math.max(maxLen, right - left + 1);
    }

    return maxLen;
}
```

**关键点：**
- Set 写法遇到重复字符时，要用 `while` 一直删到窗口里没有当前字符为止
- Map 写法里 `map.get(char) >= left` 不能省，因为 Map 里记录的是历史位置，旧字符可能已经不在当前窗口里
- `left` 只能向右移动，不能回头

#### 固定长度窗口 + 字符频率匹配
**适用场景：** 找长度固定的子串/子数组，例如异位词、排列匹配、固定长度窗口统计。

**核心思路：** 窗口长度固定为 `k`，右边加入一个元素；如果窗口长度超过 `k`，左边移出一个元素；窗口长度等于 `k` 时检查是否满足条件。

```javascript
function fixedWindow(s, target) {
    const result = [];
    const targetCount = new Array(26).fill(0);
    const windowCount = new Array(26).fill(0);
    const k = target.length;

    for (let i = 0; i < target.length; i++) {
        targetCount[target.charCodeAt(i) - 97]++;
    }

    let left = 0;
    for (let right = 0; right < s.length; right++) {
        windowCount[s.charCodeAt(right) - 97]++;

        if (right - left + 1 > k) {
            windowCount[s.charCodeAt(left) - 97]--;
            left++;
        }

        if (right - left + 1 === k && isSame(targetCount, windowCount)) {
            result.push(left);
        }
    }

    return result;
}

function isSame(arr1, arr2) {
    for (let i = 0; i < 26; i++) {
        if (arr1[i] !== arr2[i]) {
            return false;
        }
    }
    return true;
}
```

**关键点：**
- 固定窗口的长度通常由题目目标决定，例如异位词窗口长度就是 `p.length`
- 每次滑动只改变两个位置：右边新字符计数加一，左边移出字符计数减一
- 窗口超过固定长度时，必须同步更新计数表后再移动 `left`
- 如果字符集固定为小写字母，可以用长度 26 的数组；如果字符集不固定，可以用 `Map`
- 数组计数字符时，`s.charCodeAt(i) - 97` 表示把 `'a' ~ 'z'` 映射到 `0 ~ 25`

### 2.4 单调队列（滑动窗口最大值）
**适用场景：** 需要O(1)获取滑动窗口内的最大值（或最小值）

**核心思路：** 维护一个单调递减的双端队列，存下标。队头永远是当前窗口的最大值。

```javascript
function slidingWindowMax(nums, k) {
    const deque = []; // 存下标，队列保持单调递减
    const result = [];

    for (let i = 0; i < nums.length; i++) {
        // 1. 踢队尾：把所有比新元素小的踢出去（它们永远不可能是最大值）
        while (deque.length > 0 && nums[deque[deque.length - 1]] < nums[i]) {
            deque.pop();
        }

        // 2. 新下标入队
        deque.push(i);

        // 3. 踢队头：如果队头已经滑出窗口，踢掉
        if (deque[0] < i - k + 1) {
            deque.shift();
        }

        // 4. 窗口满了才记录答案
        if (i >= k - 1) {
            result.push(nums[deque[0]]);
        }
    }
    return result;
}
```

**关键点：**
- 队列存**下标**，不存值——需要下标来判断队头是否滑出窗口
- 队列始终保持**单调递减**，队头 = 当前窗口最大值
- 新元素入队前，踢掉队尾所有比它小的（它们既比新元素小，又比新元素早出窗口，永无出头之日）
- 每个元素最多入队/出队各一次，时间复杂度 O(n)

**时间复杂度：** O(n)  
**空间复杂度：** O(k)

function nextGreaterElement(nums) {
    const result = new Array(nums.length).fill(-1);
    const stack = []; // 存储索引
    
    for (let i = 0; i < nums.length; i++) {

## 5. 二叉树模式

### 5.1 递归遍历
```javascript
// 前序：根 -> 左 -> 右
function preorder(root) {
    if (!root) return;
    处理当前节点(root);
    preorder(root.left);
    preorder(root.right);
}

// 中序：左 -> 根 -> 右（BST中序是有序的！）
function inorder(root) {
    if (!root) return;
    inorder(root.left);
    处理当前节点(root);
    inorder(root.right);
}

// 后序：左 -> 右 -> 根（适合自底向上）
function postorder(root) {
    if (!root) return;
    postorder(root.left);
    postorder(root.right);
    处理当前节点(root);
}
```

function levelOrder(root) {
    if (!root) return [];
    const result = [];
    const queue = [root];
    
    while (queue.length) {
        const levelSize = queue.length;
        const currentLevel = [];

            const node = queue.shift();
            currentLevel.push(node.val);
            if (node.left) queue.push(node.left);
            if (node.right) queue.push(node.right);
        }
        result.push(currentLevel);
    }
    return result;
}

## 6. 区间合并模式

### 6.1 排序 + 贪心合并
**适用场景：** 给定一组区间，合并所有重叠的区间

**核心思路：** 按起点排序后，只需比较当前区间与结果末尾区间是否重叠

```javascript
function merge(intervals) {
    intervals.sort((a, b) => a[0] - b[0]); // 按起点升序排序
    const result = [intervals[0]];

    for (let i = 1; i < intervals.length; i++) {
        const last = result[result.length - 1];
        const curr = intervals[i];

        if (curr[0] <= last[1]) {
            // 重叠：终点取两者最大值（处理包含关系，如 [1,10] 和 [2,6]）
            last[1] = Math.max(last[1], curr[1]);
        } else {
            result.push(curr);
        }
    }
    return result;
}
```

**时间复杂度：** O(n log n)，排序主导，for 循环是 O(n)  
**空间复杂度：** O(n)

**关键点：**
- 按起点排序后，起点一定是前一个更小，合并时只需关心终点
- 终点必须取 `Math.max`，处理"大区间包含小区间"的情况（如 `[1,10]` 包含 `[2,6]`）
- 时间复杂度是 O(n log n) 而非 O(n)，因为排序这一步不可省略

### 6.2 以当前位置结尾的最优值（Kadane 模式）
**适用场景：** 求连续子数组的最大和 / 最大乘积等，当前状态只依赖前一个

**核心思路：** 遍历时维护"以当前元素结尾的最优值"，若前段是负担则丢弃重新开始

```javascript
function maxSubArray(nums) {
    let currentMax = nums[0]; // 以当前位置结尾的最大子数组和
    let maxSum = nums[0];     // 全局最大值

    for (let i = 1; i < nums.length; i++) {
        currentMax = Math.max(nums[i], nums[i] + currentMax);
        maxSum = Math.max(maxSum, currentMax);
    }
    return maxSum;
}
```

**初始化注意：** 必须用 `nums[0]` 而非 `0`，因为子数组至少选一个元素（全负数时答案是最大的负数）

**时间复杂度：** O(n)  
**空间复杂度：** O(1)

function uniquePaths(m, n) {

    for (let j = 0; j < n; j++) dp[0][j] = 1;

## 哈希表专题

### 1. 两数之和
**难度：** 简单 ⭐⭐⭐  
**完成日期：** 2026-05-25

**题目：** 给定数组和目标值，找出两个数的索引使其和为目标值。

**核心思路：**
- 边遍历边查找，用哈希表存储"已经遍历过的值和索引"
- 对于当前元素，计算 `complement = target - nums[i]`
- 在哈希表中查找 complement 是否存在

**代码：**
```javascript
var twoSum = function(nums, target) {
    const map = new Map();
    for(let i = 0; i < nums.length; i++) {
        const complement = target - nums[i];
        if(map.has(complement)) {
            return [map.get(complement), i];
        }
        map.set(nums[i], i);
    }
};
```

**时间复杂度：** O(n)  
**空间复杂度：** O(n)

**易错点：**
- ❌ 返回值顺序：应该是 `[map.get(complement), i]` 而不是 `[i, map.get(complement)]`，**先返回已经存了的**
- ❌ 先全部存入再查找 vs 边遍历边查找：两种都对，但后者更简洁

**学习心得：**
- 两种方案对比：
  - 方案A：先全部存入map，再遍历查找 - 需要两次遍历，需要判断 `!== i`
  - 方案B：边遍历边查找 - 只需一次遍历，代码更简洁
  - 结论：两种都正确，主要区别在效率和代码简洁性

### 128. 最长连续序列
**难度：** 中等 ⭐⭐⭐  
**完成日期：** 2026-06-10

**题目：** 给定一个未排序的整数数组，找出数字连续的最长序列长度，要求时间复杂度为 O(n)。

**核心思路：**

- 先用 `Set` 去重，并支持 O(1) 判断元素是否存在
- 遍历每个数字，只从 `num - 1` 不存在的数字开始统计
- 如果 `num - 1` 存在，说明 `num` 不是起点，直接跳过
- 从起点开始不断判断 `current` 是否存在，统计连续长度

**代码：**

```javascript
var longestConsecutive = function(nums) {
    const set = new Set(nums);
    let maxLen = 0;

    for (const num of set) {
        if (!set.has(num - 1)) {
            let current = num;
            let len = 0;

            while (set.has(current)) {
                current++;
                len++;
            }

            maxLen = Math.max(maxLen, len);
        }
    }

    return maxLen;
};
```

**时间复杂度：** O(n)  
**空间复杂度：** O(n)

**易错点：**
- 如果写 `let len = 1`，while 条件应写 `set.has(current + 1)`，否则会重复计算起点
- 如果写 `while (set.has(current))`，`len` 应该从 0 开始
- 不能用普通排序满足题目要求，因为排序通常是 O(n log n)

**学习心得：**
- `Map` 是键值对，适合存“值 -> 信息”；`Set` 是集合，**适合判断“元素在不在”**
- 判断起点的标准是：前一个数字 `num - 1` 不存在
- 虽然代码有 `for + while`，但每段连续序列只从起点统计一次，所以整体仍是 O(n)

### 560. 和为 K 的子数组
**难度：** 中等 ⭐⭐⭐  
**完成日期：** 2026-06-11

**题目：** 给定一个整数数组和整数 `k`，统计和为 `k` 的连续子数组个数。

**核心思路：**
- `sum` 表示从数组开头累加到当前位置的前缀和
- `map` 存的是“历史前缀和 -> 出现次数”
- 如果当前前缀和是 `sum`，目标子数组和是 `k`，就要找之前是否出现过 `sum - k`
- 因为 `sum - oldSum = k`，所以 `oldSum = sum - k`
- 每一轮必须先查历史前缀和，再把当前 `sum` 存入 `map`

**代码：**
```javascript
var subarraySum = function(nums, k) {
    const map = new Map();
    map.set(0, 1);

    let sum = 0;
    let count = 0;

    for (const num of nums) {
        sum += num;

        if (map.has(sum - k)) {
            count += map.get(sum - k);
        }

        map.set(sum, (map.get(sum) || 0) + 1);
    }

    return count;
};
```

**时间复杂度：** O(n)  
**空间复杂度：** O(n)

**易错点：**
- `map` 存的不是数组元素出现次数，而是前缀和出现次数
- 初始化 `map.set(0, 1)` 是为了统计从下标 0 开始的子数组
- 必须先查 `sum - k`，再存当前 `sum`，否则 `k = 0` 时可能多算空子数组
- 空间复杂度是 O(n)，因为最多会记录 n 个不同的前缀和，使用的是 `Map` 不是 `Set`

**学习心得：**
- `map.get(sum - k)` 的值表示有几个历史位置能作为子数组起点前面的位置
- 同一个前缀和可能出现多次，所以要存次数，而不是只存是否存在
- 一句话核心思路：用 `map` 记录历史前缀和次数，当前 `sum` 每次寻找历史里的 `sum - k`

## 数组专题

### 238. 除自身以外数组的乘积
**难度：** 中等 ⭐⭐⭐  
**完成日期：** 2026-06-14

**题目：** 给定整数数组 `nums`，返回数组 `answer`，其中 `answer[i]` 等于 `nums` 中除 `nums[i]` 之外其余元素的乘积。要求不使用除法，并在 O(n) 时间内完成。

**核心思路：**
- 每个位置的答案 = 左边所有数的乘积 × 右边所有数的乘积
- 第一遍从左到右，让 `answer[i]` 先保存 `nums[i]` 左边所有数的乘积
- 第二遍从右到左，用变量 `right` 滚动保存**当前位置右边所有数的乘积**
- 处理当前位置时，先把 `answer[i]` 乘上 `right`，再把 `nums[i]` 乘进 `right`

**代码：**
```javascript
var productExceptSelf = function(nums) {
    const n = nums.length;
    const answer = new Array(n);

    answer[0] = 1;
    for (let i = 1; i < n; i++) {
        answer[i] = answer[i - 1] * nums[i - 1];
    }

    let right = 1;
    for (let i = n - 1; i >= 0; i--) {
        answer[i] *= right;
        right *= nums[i];
    }

    return answer;
};
```

**时间复杂度：** O(n)  
**额外空间复杂度：** O(1)，输出数组 `answer` 不计入额外空间

**易错点：**
- `right` 会不断更新，所以必须用 `let`，不能用 `const`
- 第二遍必须先 `answer[i] *= right`，再 `right *= nums[i]`，否则会把当前位置自己算进去
- `answer[i]` 表示修改某个位置，不能写成 `answer *= right`

**学习心得：**
- 第一遍结束后，`answer` 暂时不是最终答案，而是左乘积数组
- 第二遍中，处理 `i` 前的 `right` 表示 `nums[i]` 右边所有数的乘积
- 优化版把输出数组复用成左乘积数组，只额外使用一个 `right` 变量，因此额外空间是 O(1)

### 287. 寻找重复数
**难度：** 中等 ⭐⭐⭐  
**完成日期：** 2026-06-15

**题目：** 给定一个包含 `n + 1` 个整数的数组 `nums`，其中每个整数都在 `1 ~ n` 范围内，找出重复的数字。要求不能修改数组，并且只能使用 O(1) 的额外空间。

**核心思路：**
- 把数组看成“下标指向 `nums[i]` 的隐式链表”
- `nums[i]` 表示从下标 `i` 下一步要跳到的下标
- 因为数字范围是 `1 ~ n`，不会跳回下标 `0`，所以 `0` 只是进入链表的起点
- 重复数字表示有两个不同下标指向同一个下标，这个位置就是环入口
- 用快慢指针先找环内相遇点，再从起点和相遇点同时出发找环入口

**代码：**
```javascript
var findDuplicate = function(nums) {
    let slow = 0;
    let fast = 0;

    do {
        slow = nums[slow];
        fast = nums[nums[fast]];
    } while (slow !== fast);

    let ptr1 = 0;
    let ptr2 = slow;

    while (ptr1 !== ptr2) {
        ptr1 = nums[ptr1];
        ptr2 = nums[ptr2];
    }

    return ptr1;
};
```

**时间复杂度：** O(n)  
**空间复杂度：** O(1)

**易错点：**
- 第一阶段不能直接写 `while (slow !== fast)`，因为初始时 `slow` 和 `fast` 都是 0，循环不会执行
- `fast = nums[nums[fast]]` 表示快指针走两步：先到 `nums[fast]`，再到 `nums[nums[fast]]`
- 第二阶段两个指针都要每次走一步，不能继续一快一慢
- `ptr2 = slow` 和 `ptr2 = fast` 都可以，因为第一阶段结束时二者已经相遇
- 用 `Set` 可以找到重复数，但额外空间是 O(n)，不符合题目 O(1) 空间要求

**学习心得：**
- 这题最难的是建模：把数组理解成“传送门”，当前位置是下标，数组值是下一步跳到哪里
- 重复数字之所以是环入口，是因为两个位置的值相同，也就是两个下标都指向同一个下标
- 快慢指针第一次相遇后，从起点和相遇点各放一个指针同速前进，会在环入口相遇

### 48. 旋转图像
**难度：** 中等 ⭐⭐⭐  
**完成日期：** 2026-06-15

**题目：** 给定一个 `n x n` 的二维矩阵 `matrix`，将图像顺时针旋转 90 度。要求原地修改矩阵。

**核心思路：**
- 顺时针旋转 90 度可以拆成两步：先沿主对角线转置，再反转每一行
- 转置时交换 `matrix[i][j]` 和 `matrix[j][i]`
- 只遍历 `j = i + 1` 到 `n - 1`，避免同一对元素交换两次
- `matrix[i].reverse()` 会把第 `i` 行原地反转

**代码：**
```javascript
var rotate = function(matrix) {
    const n = matrix.length;

    for (let i = 0; i < n; i++) {
        for (let j = i + 1; j < n; j++) {
            const temp = matrix[i][j];
            matrix[i][j] = matrix[j][i];
            matrix[j][i] = temp;
        }
    }

    for (let i = 0; i < n; i++) {
        matrix[i].reverse();
    }
};
```

**时间复杂度：** O(n²)  
**空间复杂度：** O(1)

**易错点：**
- LeetCode 要求原地修改，通常不需要 `return matrix`
- 转置时内层循环应从 `j = i + 1` 开始，否则会重复交换
- `reverse()` 是数组内置方法，会直接修改原数组
- 空间复杂度看的是同一时刻额外占用的空间，不是临时变量执行了多少次

**学习心得：**
- 二维数组中，`matrix[i]` 表示第 `i` 行，`matrix[i][j]` 表示第 `i` 行第 `j` 列
- 顺时针旋转公式是 0-based 下标下的 `[i][j] -> [j][n - 1 - i]`
- 顺时针旋转 90 度 = 先转置，再反转每一行
- 逆时针旋转 90 度也可以用类似思路：先转置，再反转每一列；或者先反转每一行，再转置

### 3. 无重复字符的最长子串
**难度：** 中等 ⭐⭐⭐  
**完成日期：** 2026-06-16

**题目：** 给定一个字符串 `s`，请找出其中不含有重复字符的最长子串的长度。

**核心思路：**
- 用滑动窗口维护一个“不含重复字符”的连续子串
- 右指针负责探索新字符
- 如果右边遇到窗口里已有的字符，左指针负责收缩窗口
- 窗口重新合法后，更新最大长度

**Set 写法代码：**
```javascript
var lengthOfLongestSubstring = function(s) {
    const set = new Set();
    let left = 0;
    let maxLen = 0;

    for (let right = 0; right < s.length; right++) {
        while (set.has(s[right])) {
            set.delete(s[left]);
            left++;
        }

        set.add(s[right]);
        maxLen = Math.max(maxLen, right - left + 1);
    }

    return maxLen;
};
```

**Map 写法代码：**
```javascript
var lengthOfLongestSubstring = function(s) {
    const map = new Map();
    let left = 0;
    let maxLen = 0;

    for (let right = 0; right < s.length; right++) {
        const char = s[right];

        if (map.has(char) && map.get(char) >= left) {
            left = map.get(char) + 1;
        }

        map.set(char, right);
        maxLen = Math.max(maxLen, right - left + 1);
    }

    return maxLen;
};
```

**时间复杂度：** O(n)  
**空间复杂度：** O(min(n, 字符集大小))

**易错点：**
- Set 写法必须用 `while` 收缩窗口，不能只用 `if` 删除一次；例如 `"abba"` 扫到第二个 `b` 时要删掉 `a` 和第一个 `b`
- Map 写法必须判断 `map.get(char) >= left`，否则会把已经不在窗口里的旧字符也当成重复字符
- `left` 只能向右移动，不能因为历史位置而回退
- 文件中如果写两个同名函数，后面的函数会覆盖前面的函数

**学习心得：**
- 右指针负责探索新字符，左指针负责修复窗口
- 右边遇到同样的字符，左指针要移到之前出现过的那个字符的下一个位置
- 滑动窗口必须始终表示原字符串中的连续区间，不能“丢掉右边字符”来跳过重复

### 438. 找到字符串中所有字母异位词
**难度：** 中等  
**完成日期：** 2026-06-16

**题目：** 给定两个字符串 `s` 和 `p`，找到 `s` 中所有 `p` 的异位词子串，返回这些子串的起始索引。

**核心思路：**
- 异位词必须和 `p` 长度相同，所以窗口长度固定为 `p.length`
- 用 `pCount` 记录 `p` 的字符频率
- 用 `windowCount` 记录当前窗口的字符频率
- 右指针加入新字符；如果窗口长度超过 `p.length`，左指针移出旧字符
- 当窗口长度等于 `p.length` 且两个计数数组完全相同时，把 `left` 加入答案

**代码：**
```javascript
var findAnagrams = function(s, p) {
    const result = [];
    const pCount = new Array(26).fill(0);
    const windowCount = new Array(26).fill(0);

    for (let i = 0; i < p.length; i++) {
        pCount[p.charCodeAt(i) - 97]++;
    }

    let left = 0;
    for (let right = 0; right < s.length; right++) {
        windowCount[s.charCodeAt(right) - 97]++;

        if (right - left + 1 > p.length) {
            windowCount[s.charCodeAt(left) - 97]--;
            left++;
        }

        if (right - left + 1 === p.length && isSame(pCount, windowCount)) {
            result.push(left);
        }
    }

    return result;
};

function isSame(arr1, arr2) {
    for (let i = 0; i < arr1.length; i++) {
        if (arr1[i] !== arr2[i]) {
            return false;
        }
    }

    return true;
}
```

**时间复杂度：** O(n)，每次比较 26 个字符频率，26 是常数  
**空间复杂度：** O(1)，两个长度为 26 的数组

**易错点：**
- `charCodeAt` 不是 `chatCodeAt`
- `p.length` 不能写成 `p,length`，后者是 JavaScript 的逗号运算符
- 窗口左边字符移出时，必须先让 `windowCount` 对应计数减一，再移动 `left`
- 如果单独保存 `length`，收缩窗口后要重新计算；更推荐直接使用 `right - left + 1`
- 如果使用 `Map` 计数，字符数量减到 0 后最好删除 key，否则比较 `Map.size` 时可能出错

**学习心得：**
- 3 题是不定长窗口，维护“无重复字符”；438 题是定长窗口，维护“字符频率和目标完全相同”
- 固定一个长度为 `p.length` 的滑动窗口，用 `windowCount` 记录字符频率，当 `windowCount` 和 `pCount` 字符频率完全相同时把 `left` 加入答案
- `s[i]` 得到字符，`s.charCodeAt(i)` 得到字符编码；数组计数时要把字符编码映射到 `0 ~ 25`

### 239. 滑动窗口最大值
**难度：** 困难  
**完成日期：** 2026-06-18

**题目：** 给定数组和窗口大小 k，返回每个滑动窗口中的最大值。

**核心思路：**
- 用单调递减双端队列维护"有可能成为最大值的候选"
- 队列存下标，队头永远是当前窗口的最大值
- 新元素入队前，踢掉队尾所有比它小的（它们既小又先出窗口，永无出头之日）
- 每次窗口滑动后检查队头是否滑出窗口

**代码：**
```javascript
var maxSlidingWindow = function(nums, k) {
    const deque = []; // 存下标，保持单调递减
    const result = [];

    for (let i = 0; i < nums.length; i++) {
        // 踢队尾：所有比新元素小的踢出去
        while (deque.length > 0 && nums[deque[deque.length - 1]] < nums[i]) {
            deque.pop();
        }
        deque.push(i);

        // 踢队头：滑出窗口的踢掉
        if (deque[0] < i - k + 1) {
            deque.shift();
        }

        // 窗口满了才记录答案
        if (i >= k - 1) {
            result.push(nums[deque[0]]);
        }
    }
    return result;
};
```

**时间复杂度：** O(n)，每个元素最多入队/出队各一次  
**空间复杂度：** O(k)

**易错点：**
- 循环条件 `i < nums.length`，不能写 `i <= nums.length`，否则越界
- 取队尾用 `deque[deque.length - 1]`，注意不要拼错 `length`
- `shift()` 不需要传参数
- 先入队，再检查队头是否越界，顺序不能颠倒

**学习心得：**
- 队列和窗口是两回事：窗口是题目定义的 k 个元素范围，队列是我们用来快速找最大值的工具
- 队列里存的是"窗口内还有资格的候选"，始终保持单调递减，所以队头就是最大值
- 暴力解每次重新扫窗口是 O(nk)；单调队列利用了"被踢出的元素永远没用"这个性质，降到 O(n)

## 11. 盛最多水的容器

**难度：** 中等  
**完成日期：** 2026-06-24  
**专题：** 双指针 - 对撞指针

**题目：** 给定高度数组，找两根柱子使之间能装最多的水。

**核心思路：**
- 对撞指针从两端出发，每次计算面积并更新最大值
- 面积由短板决定：移动较短的那一边
- 移动长板无意义：短板没变，宽度还变小，面积只会更小

**代码：**
```javascript
var maxArea = function(height) {
    let left = 0;
    let right = height.length - 1;
    let max = 0;

    while (left < right) {
        const width = right - left;
        const h = Math.min(height[left], height[right]);
        const area = width * h;

        max = Math.max(max, area);

        if (height[left] >= height[right]) {
            right--;
        } else {
            left++;
        }
    }

    return max;
};
```

**时间复杂度：** O(n)  
**空间复杂度：** O(1)

**学习心得：**
- 贪心核心：移动长板只会让宽度变小，短板还在，面积不可能变大；移动短板才有机会变大

### 15. 三数之和
**难度：** 中等  
**完成日期：** 2026-06-24  
**专题：** 双指针 - 排序 + 固定i + 对撞指针

**题目：** 给定整数数组，返回所有不重复的三元组 `[a, b, c]` 使得 `a + b + c = 0`。

**核心思路：**
- 先排序，使对撞指针可以通过移动方向判断和的大小
- 固定第一个数 `nums[i]`，用对撞指针在剩余部分找两数之和等于 `-nums[i]`
- 三处去重：`i`、`left`、`right` 各自跳过连续重复值

**代码：**
```javascript
var threeSum = function(nums) {
    const result = [];
    nums.sort((a, b) => a - b);

    for (let i = 0; i < nums.length - 2; i++) {
        if (i > 0 && nums[i] === nums[i - 1]) continue; // i 去重
        if (nums[i] > 0) break; // 最小三数已 > 0，无解

        let left = i + 1, right = nums.length - 1;
        while (left < right) {
            const sum = nums[i] + nums[left] + nums[right];
            if (sum === 0) {
                result.push([nums[i], nums[left], nums[right]]);
                while (left < right && nums[left] === nums[left + 1]) left++;   // left 去重
                while (left < right && nums[right] === nums[right - 1]) right--; // right 去重
                left++;
                right--;
            } else if (sum < 0) {
                left++;
            } else {
                right--;
            }
        }
    }
    return result;
};
```

**时间复杂度：** O(n²)  
**空间复杂度：** O(1)

**易错点：**
- 去重 `while` 不能写成 `if`：连续多个重复值时 `if` 只跳一次，会产生重复三元组
- `i > 0` 不能省：确保有前一个元素才比较，逻辑上的必要保护
- 排序是前提：无序数组中对撞指针无法判断移动方向

**学习心得：**
- 重复的 `i` 对应的搜索空间是前一个 `i` 的子集，跳过不会漏答案
- `i > 0 && nums[i] === nums[i-1]`：JavaScript 中 `nums[-1]` 返回 `undefined` 而非报错，但 `i > 0` 是逻辑正确性的必要条件

### 42. 接雨水
**难度：** 困难 ⭐⭐⭐  
**完成日期：** 2026-06-25  
**专题：** 双指针

**题目：** 给定 n 个非负整数表示柱子高度，计算能接多少雨水。

**核心思路：**
- 每个位置能存的水 = `min(左边最高, 右边最高) - height[i]`
- 双指针从两端向中间，维护 `leftMax` 和 `rightMax`
- 哪边 max 小，那边水量已由自己的 max 决定（另一边更高，min 取自己这边），可安全结算

**代码：**
```javascript
var trap = function(height) {
    let left = 0, right = height.length - 1;
    let leftMax = height[0], rightMax = height[right];
    let result = 0;

    while (left < right) {
        if (leftMax >= rightMax) {
            result += rightMax - height[right];
            right--;
            rightMax = Math.max(rightMax, height[right]);
        } else {
            result += leftMax - height[left];
            left++;
            leftMax = Math.max(leftMax, height[left]);
        }
    }
    return result;
};
```

**时间复杂度：** O(n)  
**空间复杂度：** O(1)

**易错点：**
- max 的更新在移动指针之后（为下一步准备），初始化必须包含两端元素
- `result += max - height[当前]` 结果一定非负，因为 max >= height[当前]

**学习心得：**
- 暴力 O(n²) → 预处理左右 max 数组 O(n) 空间 → 双指针 O(1) 空间，三种思路都源于同一个公式
- 双指针的核心：较小的 max 那边水量已确定，另一边不管多高都不影响结果

**题目：** 给定整数数组 `nums`，找到和最大的连续子数组，返回其最大和。

**核心思路：**
- 遍历时维护"以当前位置结尾的最大子数组和" `currentMax`
- 若前段是负担（`currentMax < 0`），就从当前数重新开始；否则接上前段

**状态转移：**
```
currentMax = Math.max(nums[i], currentMax + nums[i])
```

```javascript
var maxSubArray = function(nums) {
    let currentMax = nums[0];
    let maxSum = nums[0];

    for (let i = 1; i < nums.length; i++) {
        currentMax = Math.max(nums[i], nums[i] + currentMax);
        maxSum = Math.max(currentMax, maxSum);
    }

    return maxSum;
};
```

**时间复杂度：** O(n)  
**空间复杂度：** O(1)

**易错点：**
- 初始化必须用 `nums[0]` 而非 `0`，因为子数组至少要选一个元素（全负数时答案是最大的负数）

**学习心得：**
- 这是最简单形式的 DP：当前状态只依赖前一个状态
- 贪心理解：前段子数组如果是"负担"就丢掉，不影响找到最优解

### 56. 合并区间
**难度：** 中等  
**完成日期：** 2026-06-29  
**专题：** 区间合并 - 排序 + 贪心

**题目：** 给出一组区间，合并所有重叠的区间，返回不重叠的区间数组。

**核心思路：**
- 按起点排序，保证区间从左到右处理
- 遍历时比较当前区间与结果末尾区间：起点 ≤ 末尾终点则重叠，合并终点取 max
- 否则直接推入结果

**代码：**
```javascript
var merge = function(intervals) {
    intervals.sort((a, b) => a[0] - b[0]);
    const result = [intervals[0]];

    for (let i = 1; i < intervals.length; i++) {
        const last = result[result.length - 1];
        const curr = intervals[i];

        if (curr[0] <= last[1]) {
            last[1] = Math.max(last[1], curr[1]);
        } else {
            result.push(curr);
        }
    }
    return result;
};
```

**时间复杂度：** O(n log n)  
**空间复杂度：** O(n)

**易错点：**
- 终点合并必须用 `Math.max`，不能直接取 `curr[1]`（存在大区间包含小区间的情况）
- 时间复杂度是 O(n log n) 而非 O(n)——虽然遍历是 O(n)，但排序这一步主导了总复杂度

**学习心得：**
- 排序后起点必然有序，合并只需关心终点
- "排序让后续逻辑变简单"的代价是时间复杂度至少 O(n log n)
